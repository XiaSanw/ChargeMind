# ChargeMind 输出方向技术评审（最终版）

> **评审对象**：另一大模型的两轮输出建议 —— "5 表 + 1 模块"方案 + "新增数据维度与可提取特征"方案
> **评审依据**：嘉老师 04-24 会议纪要、实际数据质量（10942 条 stations）、现有输出界面设计
> **评审日期**：2026-04-25

## 一、总体判断

**数据哲学完全正确，输出形态需要改。融合后的方案比双方各自的原方案都更强。**

对方对数据质量的理解是准确的，我们已逐项核实。两轮讨论后达成的核心共识：

| 决策 | 内容 |
|------|------|
| 博弈模型 → 竞争定位分析 | 放弃纳什均衡，只做硬算指标，不叫博弈模型 |
| 可视化优先 | 图表是信息压缩，表格是展开层 |
| 5 维雷达图 | 地段禀赋/硬件适配/定价精准/运营产出/需求饱和度 |
| 利用率诚实标注 ⭐ | 中位数 3%，19.6% < 1%，不可作为可信参考值 |
| LLM 不生成数字 | 只叙事包装 + 异常检测 |
| 品牌维度纳入 DEMO | 数据已就绪，限定"私家车市场竞争格局" |

## 三、第一轮方案评审：「5 表 + 1 模块」

### 3.1 对方做得对的 ✅

1. **硬数据为主、LLM 为辅**——这是该方案最大的价值
2. **可信度标签系统（⭐⭐⭐/⭐⭐/⭐/⚠️）**——直接采纳
3. **功率错配分析（TVD）**——真正的运营洞察，我们现有设计缺乏
4. **竞品价格对标**——直接命中嘉老师原话"把周边两公里、五公里其他场站的充电价格做一个对比"，且数据全在
5. **LLM 异常识别**——比我们现有 mock RAG analysis 有价值得多
6. **车型-功率交叉矩阵**——功率错配的"为什么"解释层，数据全有

### 3.2 对方缺失的 ❌

对照嘉老师会议纪要：

| 嘉老师要求 | 原话 | 对方方案 |
|-----------|------|---------|
| 多维雷达图 | "是不是六边形战士啊" | ❌ |
| 趋势推演 | "未来的营收趋势推演和一个建议" | ❌ |
| 提升路径 | "降本、增效、运维、盘活" | ❌ |
| 博弈模型 | "再加一个博弈模型" | ❌ |
| 体检报告感 | "越丰富越好" | ❌ 纯表格，像数据分析报告 |

**核心矛盾**：对方方案隐含假设"硬数据 = 表格呈现"。硬数据同样可以用图表呈现。

## 四、博弈模型的重新定义：改为竞争定位分析

### 4.1 为什么改名

真正的博弈模型（纳什均衡、竞品反制推演）需要竞品成本结构和历史面板数据，我们当前只有静态快照。强求博弈模型会产出伪精确的结果。

但我们能做的硬数据竞争分析，已经比现有 `algorithm_stub`（`predicted_utilization = 0.042, is_stub: true`）有价值得多。

### 4.2 能做：硬算指标（零假设）

| 输出 | 计算方式 | 标签 |
|------|---------|------|
| 竞争基准价 | 同 grid 竞品服务费按桩数加权平均 | ⭐⭐⭐ 实测 |
| 你的价差 | 你的服务费 − 基准价 | ⭐⭐⭐ 实测 |
| 容量份额 vs 实际份额 | 你的功率占比 vs 你的充电量占比 | ⭐⭐⭐ 实测 |
| 功率错配 TVD | 供给功率分布 vs 需求功率分布 | ⭐⭐⭐ 实测 |

这四个指标回答的是同一个问题：**"为什么我的场站不行？"**

```
示例输出：
  你在网格 GXD003 装了 20.3% 的功率容量
  但只吃到 0.19% 的充电量
  你的服务费（¥0.45）比竞争基准价（¥0.32）高 41%
  你的桩型偏大——76% 是 ≥120kW 桩，但周边 95% 的车只需要 120kW 以下
```

### 4.3 能做：弹性假设推演（需诚实标注）

| 输出 | 计算方式 | 标签 |
|------|---------|------|
| 均衡利用率区间 | 假设弹性 ε ∈ [1.0, 3.0] 的份额变动区间 | ⭐⭐ 推演（标注"基于行业平均弹性假设"）|

### 4.4 做不了（需要额外数据）

- 真正的纳什均衡（缺竞品成本结构）
- 竞品反制推演（缺历史面板数据）
- 弹性系数估计（缺价格-利用率回归数据）

## 五、新增数据维度评审

### 5.1 特征 1：品牌构成矩阵 → P0，采纳

**条件**：标题限定为"私家车市场竞争格局"，不写"周边车辆品牌构成"。因为 52/93 标签有品牌，41 个无品牌标签（出租车、公交车、公务车等）在部分场站可能占大头。品牌矩阵只覆盖私家车和营运车，不能代表全量车流。

不是独立模块，嵌入到竞品对标卡片和功率建议里作为解释层——"你周边特斯拉车主占私家车 20%，他们对充电速度敏感"。

### 5.2 特征 2：电池容量集中度 → P0，与 TVD 互补

电池容量分析不替代 TVD，而是 TVD 的"为什么"解释层：

| | TVD 功率错配 | 电池容量分析 |
|---|---|---|
| 回答的问题 | 功率供给对路吗 | 为什么需要这个功率 |
| 数据类型 | 桩端 | 车端 |
| 标签 | ⭐⭐⭐ | ⭐⭐⭐ |

"65% 的车电池只有 40-60kWh，从 20% 充到 80% 需 20-30 度电，120kW 桩 12-18 分钟即可满足，你建的 360kW 超充边际效用为零"——完整叙事链。

### 5.3 特征 3：品牌充电焦虑指数 → P1，需改名改公式

**公式有三个问题**：
1. `mileage_day_km/100` 无上界，需归一化
2. SOC 和里程两个 0.5 权重无依据
3. date_type 在不同标签间不一致，跨标签聚合底数不对齐

**修改**：改名"**充电紧迫度**"，做网格内归一化排名而非绝对值。

### 5.4 特征 4：季节波动分析 → P0

额外价值：季节波动数据应作为趋势推演 LLM 的输入参数——如果冬季车流 +30%，年末趋势推演天然有上行偏置。

### 5.5 特征 5：充电行为模式分类 → 保持 P2，有结构性问题

四点分类法逻辑成立，但**分类是用标签级数据做车辆级聚类**，标签本身是聚合的，拿到的是"四种典型标签"而非"四种典型车辆"。DEMO 展示层可用，不给具体运营决策。

### 5.6 特征 6：品牌-功率匹配度 → 砍掉

5 个人为赋值的功率需求系数（特斯拉 0.7、理想 0.4...）无数据支撑。电池容量分析（特征 2）已用 14 档实测电池分布回答了同一个问题，且全是硬数据。这是电池容量分析的劣化版。

### 5.7 重新聚合的维度

| 维度 | 建议 |
|------|------|
| 半个时粒度峰谷分析 | P2——价值大但工作量大，DEMO 后做 |
| 停车/充电时长分布 | **升为 P2**——是行为分类的硬数据锚点 |
| 充电最大功率分布 | P3——power_level_mix 已近似可替代 |
| 迁移边明细 | P3——净流入/流出（一级聚合已有）够了 |
| 五维分型画像 | 暂缓——不让未完成维度阻塞已就绪数据 |

### 7.4 LLM 职责重新定义

```
旧：LLM 生成所有数据 → 幻觉和低质量输出
新：LLM 对硬算数据进行叙事包装 + 异常检测 + 竞争定位解读

示例：
  "你的功率错配分数 0.72，属于中度错配。
   你建了大量 360kW+ 超快充桩，但周边 95% 的车辆只需要 120kW 以下功率。
   65% 的车电池仅 40-60kWh，120kW 桩 12-18 分钟即可充满。
   你投入的超快充能力对这批车主边际效用为零。"
  ↑ LLM 写的叙事，但所有数字都是硬算的
```

## 九、关键共识

1. **数据哲学正确**：硬数据硬算，LLM 只叙事和异常检测，不生成数字
2. **不是博弈模型**：是竞争定位分析。硬算指标零假设，弹性区间诚实标注，纳什均衡等数据到位再议
3. **输出形态**：可视化优先（图表是信息压缩），表格作为展开层
4. **雷达图 5 维**：地段禀赋（天生）/硬件适配（可改）/定价精准（可调）/运营产出（结果）/需求饱和度（边界）
5. **品牌维度纳入 DEMO**：数据已就绪，限定"私家车市场竞争格局"，不是独立模块而是竞争定位的解释层
6. **所有分析紧扣单站**：以 grid 为分析单元，每个模块都回答"这个场站怎么样/为什么/怎么办"
7. **诚实标注**：每个数据点带可信度标签，利用率必标 ⭐ 估算，弹性区间必须标注假设条件
