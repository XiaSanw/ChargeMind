# ChargeMind 前端对接手册

> **目标读者**：前端开发  
> **最后更新**：2026-04-25  
> **核心变化**：RAG 检索新增 Chat 重排序，每条对标场站附带"为什么相似"的解释

---

## 一、三接口速览

| 接口 | 方法 | 延迟 | 说明 |
|------|------|------|------|
| `/api/extract` | POST | < 2s | LLM 解析用户自然语言 → 结构化画像 |
| `/api/enrich` | POST | < 1s | 判断缺失字段，返回下一个问题 |
| `/api/diagnose` | POST | 5-15s | 双引擎诊断（Stub + RAG + Chat 重排序）|

## 二、接口详细说明

### 2.1 POST /api/extract

**请求**：
```json
{ "user_input": "我在福田区华强北商圈，有10个快充桩，周边全是电子市场" }
```

**响应**：
```json
{
  "profile": {
    "region": "福田区",
    "business_type": ["商业区"],
    "pile_count": 10
  },
  "llm_used": true
}
```

**前端处理**：
- `llm_used: true` → 正常跳转
- `llm_used: false` → 显示提示"使用基础解析（LLM 不可用）"，仍可继续
- `error` 字段存在 → Toast 提示但继续流程

### 2.2 POST /api/enrich

**请求**：
```json
{ "profile": { "region": "福田区", "business_type": ["商业区"] } }
```

**响应**：
```json
{
  "complete": false,
  "next_question": {
    "key": "total_installed_power",
    "question": "装机总功率大约多少 kW？",
    "type": "number"
  },
  "missing_count": 5,
  "all_missing_keys": ["total_installed_power", "pile_count", ...]
}
```

**前端处理**：
- `complete: true` → 调用 `/api/diagnose`
- `type` 映射：`text`→Input, `number`→Input type=number, `select`→Select, `multi_select`→多选组件
- "跳过"按钮：从 profile 中移除当前 key，再 POST enrich

## 三、关键新字段（Chat 重排序相关）

### 3.1 similar_stations 新增字段

每条对标场站多了 3 个字段，**必须在前端展示**：

| 字段 | 类型 | 说明 | 展示位置 |
|------|------|------|---------|
| `rerank_score` | float | Chat 模型赋予的对比价值分（1.0=最佳）| 对标卡片上作为"推荐度" |
| `similarity_reason` | string | 为什么这个场站值得对标 | 对标卡片副标题或 hover tooltip |
| `key_comparison` | string | 最关键的指标对比 | 卡片上高亮显示 |
| `reranked_by` | string | `"chat"` 或 `"vector"` | Chat 重排成功时显示"AI 智能匹配"Badge |

### 3.2 rerank_info 字段

```json
{
  "used": true,           // 是否使用了 Chat 重排序
  "method": "chat",       // "chat" | "vector"
  "candidate_count": 8,   // 粗检索候选数
  "selected_count": 5,    // 精排选中数
  "error": null           // 降级时有错误信息
}
```

**前端处理**：
- `method: "chat"` → 对标区域显示标题"AI 智能匹配 Top-5"
- `method: "vector"` → 显示"向量匹配 Top-5"（降级态，不展示 similarity_reason）

## 四、TypeScript 类型更新

在现有 `dashboard.ts` 基础上补充：

```typescript
// 相似场站（含重排序字段）
export interface SimilarStation {
  station_id: string;
  document: string;
  metadata: StationMetadata;
  similarity_score: number;   // 原始向量相似度
  rerank_score: number;       // Chat 重排分数（1.0=最佳）
  reranked_by: 'chat' | 'vector';
  similarity_reason: string;  // 为什么值得对标
  key_comparison: string;     // 关键指标对比
}

export interface StationMetadata {
  station_name: string;
  region: string;
  business_type: string;
  total_installed_power: number;
  avg_daily_energy_kwh: number;
  avg_utilization: number;
  peak_hour: string;
  has_timeseries_data: boolean;
}

export interface RerankInfo {
  used: boolean;
  method: 'chat' | 'vector';
  candidate_count?: number;
  selected_count?: number;
  error?: string;
}

export interface RAGResult {
  similar_stations: SimilarStation[];
  analysis: string;           // Markdown
  rerank_info: RerankInfo;
}

export interface DiagnoseResponse {
  profile: StationProfile;
  algorithm: AlgorithmResult;
  rag: RAGResult;
  report: Report;
}
```

## 五、前端展示建议

### 5.1 对标场站卡片

每条相似场站用一张卡片展示，核心信息层次：

```
┌──────────────────────────────────────────┐
│  🔍 AI 智能匹配 · 推荐度 95%              │  ← rerank_score × 100
│  ────────────────────────────────────────│
│  📍 华强北赛格广场充电站                    │  ← station_name
│    福田区 · 商业区                         │  ← region + business_type
│                                           │
│  💡 同属福田商业区，硬件配置高度一致，       │  ← similarity_reason
│    但利用率比你高 3 倍，且有时序数据支撑     │
│                                           │
│  ⚡ 你的利用率 4.2% vs 它的 12.5%          │  ← key_comparison（高亮）↑↑↑
│                                           │
│  ┌─────────┬─────────┬─────────┐          │
│  │ 装机功率 │ 日均充电 │ 利用率  │          │  ← 关键指标行
│  │ 420kW   │ 1200度  │ 12.5%  │          │
│  └─────────┴─────────┴─────────┘          │
│                                           │
│  📊 真实数据 Badge  ✓                     │  ← has_timeseries_data
└──────────────────────────────────────────┘
```

### 5.2 降级态处理

当 `rerank_info.method === "vector"` 时：
- 隐藏 `similarity_reason` 和 `key_comparison` 字段
- 标题改为"向量匹配 Top-5"
- 不显示推荐度评分
- 若 `rerank_info.error` 存在，卡片底部显示一行小字"AI 精排暂时不可用，显示基础匹配结果"

### 5.3 加载文案

| 阶段 | 时间 | 文案 |
|------|------|------|
| 解析画像 | 0-1s | 正在解析场站信息... |
| 向量检索 | 1-3s | 正在搜索 10,942 个场站数据... |
| Chat 精排 | 3-8s | AI 正在为你精选最值得对标的场站... |
| 生成报告 | 8-15s | 正在生成诊断报告... |

## 六、错误处理

| 场景 | 前端行为 |
|------|---------|
| extract 返回 `llm_used: false` | Toast "LLM 忙，已用基础解析"，继续流程 |
| enrich 网络超时 | 重试按钮，不丢失已填 profile（localStorage 兜底）|
| diagnose 超时（> 30s）| 显示 ErrorFallback + "重新诊断"按钮 |
| diagnose 返回 `rerank_info.method: "vector"` | 正常展示但隐藏 AI 匹配标识 |
| diagnose 返回空 similar_stations | 算法面板正常展示，RAG 面板显示"暂未找到对标场站" |
