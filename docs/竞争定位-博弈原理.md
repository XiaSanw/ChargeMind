# 竞争定位分析的博弈原理

> 面向评委的技术答辩材料。解释我们的模型在做什么、为什么这样做、边界在哪里。

---

## 一、我们在做什么：Bertrand 价格竞争框架

充电场站在同一个 grid 内竞争同一批车流。经济学中这叫 **Bertrand 竞争**（价格竞争，产品有差异化）：

```
同 grid 车流池
    ↓
车主选择：哪个场站充电？
    考虑因素：价格、距离、功率、排队时间...
    ↓ 简化 →
核心变量：你的价格 vs 竞品价格 → 影响你的利用率
```

每个场站的选择变量是**服务费定价**。如果所有人都定同一个价，各自按容量份额分配车流。一旦有人降价，就从他家"偷"份额。

## 二、核心公式：线性弹性需求

```
util = base_util × (1 + ε × (P_benchmark - P_my) / P_benchmark)
```

| 符号 | 含义 | 来源 |
|------|------|------|
| `base_util` | 如果你定价=基准价时的理论利用率 | 容量份额理论值（你占 grid 多少桩，理论上就该分多少车流） |
| `P_my` | 你的服务费 | 真实数据（¥/kWh） |
| `P_benchmark` | 竞品服务费加权均价 | 同 grid 竞品真实价格，按桩数加权 |
| `ε` | 价格弹性 | **行业文献（1.5-2.5），我们最大的假设** |

**解读**：
- 你定价 = 基准价 → 利用率 = base_util（公平份额）
- 你降价 10% → 利用率提升 ε×10%（弹性 2.0 → +20%）
- 你涨价 10% → 利用率下降 ε×10%（弹性 2.0 → -20%）

## 三、为什么是窄区间 [1.5, 2.5] 而不是单点

评委可能问："为什么选弹性 2.0？为什么不直接给一个数？"

**回答逻辑**：

1. **弹性不是我们能测的**。真实的充电需求弹性需要 A/B 实验或面板回归（同一场站调价前后对比）。我们没有这种数据。
2. **但我们有行业文献支撑**。充电服务是相对同质化的商品，价格弹性一般在 1-5 之间。常见估计在 1.5-2.5。
3. **区间比单点诚实**。给 `[6%, 10%]`（弹性 1.5-2.5）比对 `~8%（弹性 2.0）` 更好防追问——"为什么是 2.0"无答案，"文献在 1.5-2.5 之间"有出处。
4. **窄区间（2 倍以内）避免了"什么都有可能"的模糊感**，同时保留了对不确定性的诚实。

## 四、和"纳什均衡"的区别——诚实的边界

评委可能问："这是纳什均衡吗？"

**回答：不是，我们没有声称是。**

| | 纳什均衡（做不到） | 我们的竞争定位分析（做到了） |
|---|---|---|
| 需求 | 需要知道**需求函数** Q(P_all) | 只假设**线性弹性关系**，不精确但方向对 |
| 成本 | 需要知道每个竞品的**真实成本结构** | 不需要成本数据 |
| 均衡解 | 解联立方程，找到无改进动机的价格向量 | 算"如果你是市场中位数价格，你会在哪" |
| 输出 | "均衡服务费是 ¥0.32/度" | "基准价 ¥0.32，你 +41%，份额在流失" |
| 可信度 | 做不到（无成本数据 + 无面板回归） | ⭐⭐⭐（基准价、价差是硬数据） |

我们输出的是**竞争基准价**——同 grid 竞品服务费的桩数加权均价——这是**观测值**，不是**均衡解**。

## 五、三层输出的博弈含义

```
Layer 1: 诊断层 ⭐⭐⭐ — 纯观测，零假设
  "你在 grid 中占 20% 容量，只捕获 12% 车流，价格高 41%"
  → 这不是博弈，这是会计。不需要辩护。

Layer 2: 推演层 ⭐⭐ — 弹性假设
  "如果弹性在 1.5-2.5，你的均衡利用率约 [5%, 8%]"
  → 有假设，但假设范围来自文献。窄区间（2x）防止了"啥都可能"。

Layer 3: 建议层 ⭐⭐ — 公式透明
  "降价 ¥0.05 → 预计弹性 2.0 下利用率提升 ~X%"
  → 公式公开，评委可以自己换弹性值重算。透明 = 可信。
```

**核心防守策略**：让评委看到你是**把不确定性诚实展示出来**，而不是藏在一个看起来精确的数字后面。

## 六、评委常见追问及应答

### Q1: "你这个弹性 1.5-2.5 哪来的？"

**A**: 充电行业文献中价格弹性估计范围。我们取了中间偏保守的区间。并且我们把弹性作为**透明参数**——任何有更好数据的人可以代入自己的弹性值重算。我们展示的是方法论，不是断言。

### Q2: "为什么不直接用实际利用率算？"

**A**: 当前数据的利用率（中位数 3%）来自不精确的估算模型，用它做基准会放大噪声。我们选择用**容量份额**——你在 grid 中装了多少桩——作为理论基准，这个数据是实测的、干净的。

### Q3: "如果竞品也调价呢？你的模型不就失效了？"

**A**: 对。静态弹性模型不考虑策略互动（我降价→他跟进→我再降）。这是当前数据条件下的**静态快照分析**，不是动态博弈模拟。做真正的动态博弈需要：
- 每个场站的成本函数（至少电力成本 + 地租）
- 历史价格-需求面板数据（估计交叉弹性）
- 多期博弈求解

这些是 Demo 之后课题组的后续工作。

### Q4: "你这不就是价差分析吗，博弈在哪里？"

**A**: 博弈体现在**弹性如何将价差转化为份额变化**。纯价差分析只告诉你"你比隔壁贵 4 毛"，不告诉你这意味着多少份额流失。弹性桥梁的价格→份额映射才是博弈内核：**因为博弈，所以价差有后果**。
