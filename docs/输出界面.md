# ChargeMind 报告输出界面设计

> **定位**：可视化驾驶舱 + 精简文字，不是一段长 Markdown  
> **数据来源**：后端硬算 + LLM 叙事包装 → 前端渲染图表  
> **核心原则**：硬数据硬算，LLM 只叙事。每个数据点带可信度标签（⭐⭐⭐实测 / ⭐⭐推演 / ⭐估算 / ⚠️异常）。
> **最后更新**：2026-04-25（5 维雷达 + 称号系统 + 新增数据模块）

## 输出要求

你必须输出一个严格的 JSON 对象（不要 Markdown，不要代码块标记），结构如下：

{
  "dashboard": {
    "headline": "一句话痛点诊断，必须量化差距",
    "overall_score": 0-100的整数,
    "radar": {
      "维度名": { "score": 0-100, "comment": "一句话诊断" }
    },
    "scoring_logic": "评分计算逻辑简述"
  },
  "kpi_cards": [
    { "label": "...", "value": "...", "trend": "up/down/flat", "benchmark": "...", "detail": "..." }
  ],
  "benchmark": {
    "labels": ["你的场站", "对标1", ...至少4个],
    "metrics": [
      { "key": "...", "name": "...", "unit": "...", "values": [数值, ...] }
    ]
  },
  "paths": [
    {
      "title": "...",
      "category": "成本优化/效率提升/博弈调价/资产盘活",
      "annual_gain": "万元数值或null（无精确模型时不给）",
      "effort": "low/medium/high",
      "trust": "⭐⭐⭐/⭐⭐/⭐/⚠️",
      "calculation": "透明公式或null（被追问时能讲清楚来源）",
      "detail": "一句话说明逻辑"
    }
  ],
  "detail_text": "Markdown详细分析（唯一文字区）"
}

## 约束

1. **五维雷达**：固定五个维度——地段禀赋、硬件适配、定价精准、运营产出、需求饱和度。每项给出 0-100 评分和一句有画面感的诊断。
2. **KPI 卡片**：固定四张——均衡利用率区间、年收益预估、竞争基准价差、高峰时段。每张包含趋势方向和可信度标签。
3. **竞品价格对标（#8 简化版）**：min/avg/max 三档 + 峰谷比（spread_ratio）。不做分时段对齐，公式透明。
4. **标杆对比**：至少包含 4 个对标对象（含区域均值），至少 3 个指标维度。
5. **季节趋势推演**：不画三线折线图（无精确收益模型），改为 LLM 基于季节数据做方向性判断（上行/下行/平稳），20 字以内，禁止数字。
6. **提升路径**：至少 3 条，按 effort 排序（low → medium → high）。
7. **数字三层策略**（关键约束）：
   - **诊断层**（⭐⭐⭐硬数据）：给点估计，被追问能当场推导（TVD 0.871、价差 ¥0.13、份额差距 +6.9%）
   - **推演层**（⭐⭐推演）：给**窄区间**+强标注，弹性假设范围 1.5-2.5（如 `[6%, 10%]`），波动控制在 2 倍以内
   - **建议层**（⭐⭐推演）：只给"公式透明"的数字，不给概率/无依据收益
     - ✅ 可以给：峰谷优化 ¥3.2万（公式：240度×40%×¥0.9×365）
     - ❌ 不给："概率 70%""预计提升利用率 8%"（无历史干预数据）
     - null：无精确模型时给"建议方向"不给数字
8. **聚焦直接回报**：不考虑人员成本、运维成本。
9. **detail_text 精简**：作为唯一文字区也控制在 500 字以内，点到为止。

## 五维度评分逻辑参考

- **地段禀赋**：grid 车流量排名 + 车型丰富度 + 平均 SOC + 净流入车流
- **硬件适配**：功率档位分布 vs 周边车型功率需求（TVD 错配分数）+ 电池容量匹配度
- **定价精准**：各时段价格 vs 同 grid 竞品均价 + 价差百分比
- **运营产出**：利用率 + 度电收益 + 峰谷均衡度（⚠️ 利用率标注⭐估算）
- **需求饱和度**：网格日均充电需求 / (装机功率 × 有效运营小时)

### 称号系统（8 种典型形状）

| 形状 | 称号 | 判断逻辑 |
|------|------|---------|
| 全高位均衡 | 六边形战士 | 5 维都在前 30% |
| 地段突出型 | 含着金钥匙出生 | 地段 > 80 分但产出 < 40 分 |
| 硬件过剩型 | 大炮打蚊子 | TVD 错配 > 0.5 |
| 定价激进型 | 价格屠夫 | 价格低于竞品 20%+ |
| 躺平型 | 佛系充电站 | 全维度在 40-60 之间 |
| 错配型 | 穿错鞋的运动员 | 硬件与需求不匹配但产出尚可 |
| 增长型 | 风口上的猪 | 当前产出低但需求饱和度 > 80 分 |
| 内卷型 | 红海幸存者 | 竞争激烈 + 价格被压 + 饱和度低 |

> DEMO 阶段先做 4 种核心称号：六边形战士、含着金钥匙出生、大炮打蚊子、佛系充电站。

import { BarChart, Bar, XAxis, YAxis, CartesianGrid, Tooltip, Legend, Cell, ResponsiveContainer } from 'recharts';

interface Props {
  benchmark: Benchmark;
  selectedMetric: string;
  onMetricChange: (key: string) => void;
}

export function BenchmarkChart({ benchmark, selectedMetric, onMetricChange }: Props) {
  const metric = benchmark.metrics.find(m => m.key === selectedMetric)!;
  const chartData = benchmark.labels.map((label, i) => ({
    name: label,
    value: metric.values[i],
    isUser: i === 0,
  }));

  return (
    <div>
      {/* Tab 切换 */}
      <div className="flex gap-2 mb-4">
        {benchmark.metrics.map(m => (
          <button
            key={m.key}
            onClick={() => onMetricChange(m.key)}
            className={`px-3 py-1 rounded text-sm ${m.key === selectedMetric ? 'bg-primary text-primary-foreground' : 'bg-muted'}`}
          >
            {m.name}
          </button>
        ))}
      </div>
      <ResponsiveContainer width="100%" height={300}>
        <BarChart data={chartData}>
        <CartesianGrid strokeDasharray="3 3" stroke="var(--border)" />
        <XAxis dataKey="name" tick={{ fill: 'var(--foreground)', fontSize: 12 }} />
        <YAxis tick={{ fill: 'var(--foreground)', fontSize: 12 }} />
        <Tooltip />
        <Bar dataKey="value" name={metric.name} unit={metric.unit}>
          {chartData.map((entry, i) => (

          ))}
        </Bar>

import { LineChart, Line, XAxis, YAxis, CartesianGrid, Tooltip, Legend, ResponsiveContainer } from 'recharts';
