# AI驱动充电桩智能诊断平台 - 演示版需求文档 v0.1

## 1. 项目定位

将已有的场站数据盈利模型能力，封装成一个可演示的 AI 咨询工具。

**核心价值**：针对单个充电场站，输入场站信息，即可输出收益预测与购电优化建议，最终由 LLM 生成可落地的「降本增效」分析报告。

---

## 2. 系统架构（三层）

```
用户输入（自然语言/表单）
    ↓
【LLM 提取层】非结构化数据 → 结构化场站参数
    ↓
【核心算法层】结构化参数 → 收益预测 + 购电优化方案
    ↓
【LLM 迭代/生成层】基于收益和优化数据，生成降本增效建议
    ↓
输出最终诊断报告
```

### 2.1 各层职责

| 层级 | 职责 | 当前阶段实现方式 |
|------|------|------------------|
| LLM 提取层 | 从用户描述中提取场站关键参数（桩数、电价、利用率等） | 离线规则模拟 + 预留真实 LLM API 接口 |
| 核心算法层 | 根据参数计算当前收益、优化后收益、IRR、购电策略等 | 内置 Python 模拟器，生成合理数据 |
| LLM 生成层 | 基于算法结果生成言之有物的降本增效建议 | 离线模板模拟 + 预留真实 LLM API 接口 |

---

## 3. MVP 目标

搭建一个**可在 Windows 上直接运行的 EXE 演示程序**，完成以下闭环：

1. 用户在界面中输入一段场站描述文字
2. 系统自动提取关键参数并展示
3. 系统模拟算法运行，输出收益预测和购电优化数据
4. 系统生成结构化的降本增效分析报告

**当前阶段不追求**：
- 接入真实算法模型（用模拟器替代）
- 必须调用真实 LLM API（内置离线演示逻辑，预留接口）
- 文件上传解析（PDF/Excel/截图）

## 4. 演示示例话术

用户可在输入框中粘贴或输入以下内容：

> 我们在昆明市盘龙区有一个充电站，名字叫盘龙快充站。场地有20个120kW的直流快充桩，目前日均充电量约3000度，利用率在35%左右。当前电价是峰段1.2元、平段0.9元、谷段0.6元。场地月租金3万元，有3个运维人员。周边3公里内有5个竞品充电站。主要客户是网约车司机。

### 4.1 期望提取的结构化参数

```json
{
  "station_name": "盘龙快充站",
  "location": "昆明市盘龙区",
  "pile_count": 20,
  "pile_power_kw": 120,
  "daily_kwh": 3000,
  "utilization_rate": 0.35,
  "peak_price": 1.2,
  "flat_price": 0.9,
  "valley_price": 0.6,
  "monthly_rent": 30000,
  "staff_count": 3,
  "nearby_competitors": 5,
  "main_customer": "网约车司机"
}
```

## 5. 算法模拟器输出规范

基于提取的参数，模拟器应输出以下核心指标：

| 输出项 | 说明 | 示例值 |
|--------|------|--------|
| `current_annual_profit` | 当前年收益（万元） | -18.5 |
| `optimized_annual_profit` | 优化后年收益（万元） | +12.3 |
| `profit_improvement` | 收益提升（万元） | 30.8 |
| `current_irr` | 当前 IRR | -5.2% |
| `optimized_irr` | 优化后 IRR | 8.7% |
| `optimized_purchase_strategy` | 优化购电策略 | 谷段购电比例从30%提升至55%，峰段从40%降至15% |
| `target_utilization` | 目标利用率 | 55% |

---

## 6. 报告输出要求

最终报告需包含以下板块，内容需"言之有物"：

1. **场站基本信息摘要**：总结输入的场站概况
2. **当前盈利诊断**：基于模拟数据指出当前亏损/盈利瓶颈
3. **降本建议**：
   - 购电策略调整（峰谷平比例优化）
   - 运营成本控制（人员、租金谈判等）
4. **增效建议**：
   - 引流策略（针对网约车司机的定向运营）
   - 定价策略（动态调价、会员体系）
   - 设备利用率提升（错峰引导、桩位优化）
5. **优化效果预期**：量化优化前后的收益对比

## 7. 技术选型

| 模块 | 选型 | 理由 |
|------|------|------|
| 桌面应用框架 | PySide6 | 现代、功能强、PyInstaller 打包稳定 |
| 打包工具 | PyInstaller | 可输出单 EXE 文件，适合 Windows 演示 |
| LLM | Kimi (Moonshot AI) | 团队已有使用习惯，代码中预留 API 接口 |
| 后端逻辑 | Python | 与模型端同栈，便于后续替换真实算法 |
