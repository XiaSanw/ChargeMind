# 充电桩智能诊断 — 迭代手册

> 从当前 Demo 到理想产品的渐进路径。每个迭代独立可交付、可演示。

---

## 当前状态（Demo v1）

```
固定文本 → LLM提取参数 → 黑箱公式算优化 → LLM生成报告
```

**能做到的：** 一段话进去，一份报告出来，数字基本自洽
**做不到的：** 没有现实评估层、没有外部因素分析、假设全部硬编码

---

### 迭代 2：报告结构升级为三层（3-5天）

**目标：** 输出从"单层优化方案"升级为"现实评估 → 潜力空间 → 提升路径"

#### 2a. 改造 `diagnosis.py` 算法

```python
def diagnose(params: dict) -> dict:
    return {
        "reality": {          # 第一层：现实评估
            "annual_kwh": ...,
            "annual_revenue": ...,
            "utilization_rate": ...,
            "peak_valley_ratio": ...,
        },
        "potential": {         # 第二层：潜力空间
            "max_uplift_kwh": ...,
            "confidence": "中",
            "external_factors": [...],
        },
        "actions": [...],      # 第三层：提升路径（保留现有结构，增加前提条件和风险）
        "assumptions": [...],
    }
```

#### 2b. 改造报告 Prompt

- REPORT_PROMPT 按三层结构重写六个章节
- 每层对应报告的独立段落
- 潜力分析部分要求 LLM 标注置信度

#### 2c. UI 适配

- C 区（诊断数据）拆成三个折叠面板
- 现实评估展示核心指标卡片
- 潜力空间展示上下限区间
- 提升路径保留当前 action 列表样式

**交付物：** 三层结构报告，算法仍为规则公式但结构完整

---

### 迭代 3：外部因素接入（5-7天）

**目标：** 第二层"潜力空间"从硬编码变为数据驱动

| 能力 | 实现方式 |
|------|----------|
| 周边场站分析 | 输入地址 → 调用地图 API 获取周边充电站 POI |
| 竞争格局 | 根据 POI 数量、距离、功率估算竞争强度 |
| 需求侧分析 | 接入区域充电需求数据（或用 LLM 推理） |

#### 改动范围
- 新增 `core/external_data.py`：封装地图 API / POI 查询
- `core/diagnosis.py`：潜力分析部分改为参数化计算
- `core/kimi_api.py`：EXTRACT_PROMPT 增加地址解析字段
- `core/worker.py`：pipeline 增加一步外部数据获取

**交付物：** 输入地址后自动分析周边竞争，潜力评估有数据支撑

---

### 迭代 4：输入精度升级（3-5天）

**目标：** 从"一段话"升级为"结构化输入"

两个方向（可选其一或并行）：

**方案 A：标准化报表输入**
- 左栏 A 区改为表单界面（QFormLayout）
- 用户直接填字段，跳过 LLM 提取步骤
- 保留"自然语言输入"作为备选 tab

**方案 B：问答式引导输入**
- 多轮对话引导用户逐步提供信息
- LLM 判断信息是否完整，缺失则追问
- 适合不熟悉行业术语的用户

**交付物：** 更精准的输入，减少 LLM 提取错误

---
