# OpenGridWorks 项目分析文档

> 基于公开信息整理的 OpenGridWorks (opengridworks.com) 网站搭建思路、目标人群与实现手段分析。
> 数据截止：2026年4月

---

## 一、项目概述

**OpenGridWorks** 是一个专注于**电力基础设施数据可视化**的交互式地图平台，由 **Merit Order Energy, LLC** 运营，创始人 **Brian Bartholomew**。

核心定位：将分散的公开能源数据聚合到一张地图上，让用户能够直观探索发电厂、输电网络、数据中心、海底电缆等关键电力基础设施的空间分布与属性信息。

## 三、搭建思路

### 3.1 核心设计原则

1. **数据公开，免费使用**
   - 所有底层数据均来自政府公开数据集和开放许可的第三方数据
   - 不向用户收费，无需注册即可浏览基础地图

2. **轻量前端，重数据层**
   - 前端只做渲染和交互，复杂的数据处理在服务端或预构建阶段完成
   - 地图瓦片和数据查询走 CDN / 专用地图服务

3. **多层叠加，按需开关**
   - 不同数据类型作为独立的地图图层（Layer）
   - 用户通过 URL 参数或 UI 面板控制显示哪些图层
   - 示例 URL：`layers=tx,datacenters,hpoints,rowTx,rowSubs`

4. **移动端优先的响应式设计**
   - 支持触摸手势（Swipe up for filters）
   - 窄屏设备自动调整面板布局

## 四、技术栈

| 层级 | 技术/服务 | 说明 |
|------|-----------|------|
| **部署平台** | **Vercel** | 前端托管 + Serverless Functions，全球 CDN 加速 |
| **前端框架** | **Astro** | 静态站点生成，HTML 属性 `data-astro-cid-*` 可识别 |
| **地图渲染** | **Leaflet / Mapbox GL JS** (推测) | 交互式地图核心，支持矢量瓦片和 GeoJSON 叠加 |
| **底图瓦片** | **CARTO + OpenStreetMap** | 基础地理底图，Attribution 中明确标注 |
| **数据存储** | **PostGIS / 专用瓦片服务** (推测) | 地理空间数据存储和瓦片切片 |
| **矢量数据源** | **Mapbox Vector Tiles / PBF** (推测) | 输电线路、变电站等复杂几何数据的高效传输 |
| **样式方案** | 响应式设计 + 暗色/亮色主题 | `prefers-color-scheme` 媒体查询支持 |
| **域名/SSL** | Vercel 自带 | `opengridworks.com` |

## 五、数据层设计

### 5.1 数据来源总览

OpenGridWorks 的核心竞争力不是技术，而是**数据的整合与清洗能力**。数据全部来自公开渠道：

| 数据类型 | 来源 | 许可证 | 更新频率 |
|----------|------|--------|----------|
| **美国发电厂属性与容量** | EIA (能源信息署) EIA-860M | 公共领域 | 月度 |
| **输电线路、变电站、天然气管道** | HIFLD (国土基础设施数据层) | 公共领域 | 不定期 |
| **全球发电厂** | Global Energy Monitor (煤电/气电/风电/光伏追踪器) | CC BY 4.0 | 季度 |
| **美国数据中心位置** | IM3 / PNNL / DOE | ODbL 1.0 | 不定期 |
| **前沿 AI 数据中心** | Epoch AI Frontier Data Centers | CC BY 4.0 | 持续更新 |
| **计划中的输电项目** | OurGridFuture | 需引用 | 2026年3月版 |
| **海底电缆与登陆点** | TeleGeography Submarine Cable Map | CC BY-SA 4.0 | 持续更新 |
| **水资源风险** | WRI Aqueduct 4.0 | CC BY 4.0 | 年度 |
| **充电桩位置** | AFDC / DOE (替代燃料数据中心) | 公共领域 | 持续更新 |
| **基础地图与电力基础设施几何** | OpenStreetMap power layer | ODbL 1.0 | 社区持续更新 |

### 5.2 数据处理流程（推测）

1. **采集**：定期从各源头下载原始数据（CSV、Shapefile、GeoJSON、API）
2. **清洗**：统一坐标系（WGS84）、处理空值、去重、字段映射
3. **空间化**：将表格数据转换为地理点/线/多边形
4. **切片**：将大规模矢量数据切分为地图瓦片（MVT/PBF），便于前端高效渲染
5. **聚合**：发电厂按坐标聚类（Clustering），避免大量重叠点导致性能问题
6. **分发**：通过 CDN 或地图服务接口提供给前端

### 5.3 数据存储位置

- **原始数据**：分散在各政府/机构服务器（EIA.gov、HIFLD 等）
- **处理后数据**：推测托管在 Vercel 边缘网络或专用地图服务上
- **用户侧**：无本地持久化，纯浏览器端渲染；URL 参数保存当前视图状态

## 六、前端实现细节

### 6.1 URL 状态管理

地图视图状态通过 URL Query Parameters 保存，便于分享和回溯：

```
/power-plants?layers=tx,datacenters,hpoints,rowTx,rowSubs&panel=closed
```

| 参数 | 含义 |
|------|------|
| `layers` | 激活的图层列表，逗号分隔 |
| `lat` / `lng` / `z` | 地图中心坐标和缩放级别 |
| `tech` | 发电技术类型筛选 |
| `bubbleScale` | 气泡大小缩放系数 |
| `panel` | 侧面板开关状态 |

### 6.2 图层系统

每个数据类型对应独立的地图图层：

- `tx` — Transmission lines (输电线路)
- `datacenters` — Data centers (数据中心)
- `hpoints` — 发电厂热力/点位
- `rowTx` — Right-of-way transmission (输电走廊)
- `rowSubs` — Substations (变电站)
- `fiberLanding` — 光纤/海底电缆登陆点
- 其他：water risk, EV charging, planned transmission...

### 6.3 交互设计

- **桌面端**：左侧/右侧滑出面板，鼠标悬停显示详情
- **移动端**：底部抽屉式面板，上滑展开筛选器
- **气泡图（Bubble Map）**：发电厂按容量大小显示不同直径的圆点
- **聚类（Clustering）**：高密度区域自动聚合为计数气泡
- **图例**：动态显示当前激活图层的颜色/符号含义

## 七、商业模式与合规

### 7.1 商业模式

- **当前**：完全免费，无广告，无付费墙
- **潜在方向**：
  - 高级 API 访问（批量数据导出）
  - 企业定制报告
  - 私有数据层叠加（付费客户上传自有数据）

### 7.2 法律合规

- **免责声明**：明确声明数据仅供参考，不构成投资/运营/交易建议
- **数据归属**：每个数据集单独标注来源和许可证，/attribution 页面完整列出
- ** robots.txt**：可能限制爬虫（Terms 中明确禁止 scraping）
- **隐私政策**：收集最小化用户数据（匿名分析可能用 Vercel Analytics）

## 八、实现手段总结

| 维度 | 实现手段 |
|------|----------|
| **快速启动** | Astro 静态站点 + Vercel 托管，零服务器运维 |
| **地图性能** | 矢量瓦片 + 前端聚类，支撑数万级点位 |
| **数据 freshness** | 自动化 ETL 流水线，定期同步各数据源 |
| **SEO/分享** | URL 状态序列化，每个视图都是可分享的链接 |
| **成本控制** | 使用公开免费数据 + Vercel 免费/低价 tiers |
| **可扩展性** | 图层插件化架构，新增数据源只需增加一层配置 |

## 九、可借鉴的亮点

1. **"Public data for public use" 的定位** — 降低法律风险，获得社区 goodwill
2. **数据源透明化** — /attribution 页面完整列出所有数据来源和许可证，建立信任
3. **URL 即状态** — 无需后端会话，每个链接都是精准定位的视图
4. **多层叠加架构** — 不同数据类型解耦，按需加载，性能可控
5. **移动端友好** — 能源数据可视化不是只有桌面端才能做好
