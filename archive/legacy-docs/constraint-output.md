# 输出诊断约束

> 本文档定义充电场站智能诊断报告的输出结构、数据口径与约束条件。
> 基于 2026-04-14 与业务方讨论确认。

---

## 1. 分析范围

### 纳入
- 场站全年充电量
- 充电收益（电费差价 + 服务费收入）
- 场站直接运营回报（return）

### 排除
- 人员工资、社保等人力成本
- 设备折旧、维修等运维成本
- 管理费用、行政开支等间接成本

> 约束理由：聚焦场站运营的直接经济回报，排除企业层面的成本结构差异。

---

## 2. 报告三层结构

### 第一层：现实评估

| 维度 | 说明 |
|------|------|
| 数据来源 | 场站订单数据、充电记录 |
| 输出内容 | 全年充电量、收益、利用率、峰谷分布 |
| 核心指标 | 年充电量(kWh)、年充电收入(元)、桩均利用率(%)、峰谷充电占比 |
| 约束 | 纯数据驱动，不做假设，不做预测 |

### 第二层：潜力空间测算

| 维度 | 说明 |
|------|------|
| 分析方向 | 在现实评估基础上，量化增长天花板 |
| 外部因素 | 周边场站数量与布局、规模化车队覆盖、车辆停驶时间分布、区域充电需求趋势 |
| 输出内容 | 潜在提升空间的上下限、实现概率或置信区间 |
| 约束 | 必须标注假设条件和数据来源，区分"高/中/低"置信度 |

### 第三层：提升路径建模

| 路径类型 | 机制 | 预期效果 |
|----------|------|----------|
| 降本 | 参与电力现货市场交易 | 降低购电均价 |
| 降本 | 与电网/运营商签订补充协议 | 锁定优惠电价 |
| 增效 | 引流机制（定价策略、合作车队、会员体系） | 提升充电利用率 |
| 增效 | 时段优化（夜间引流、错峰调价） | 提升谷段利用率 |

每条路径需输出：
- 预期利润增量（万元/年）
- 实施前提条件
- 风险与不确定性说明

---

## 3. 数据输入约束

### 当前（Demo 阶段）
- 输入方式：自然语言描述 → LLM 提取结构化参数
- 精度：粗糙，无行业先验约束
- 适用场景：快速验证输出结构

### 目标（迭代方向）
- 输入方式：标准化报表格式 或 结构化问答
- 补充数据：场站地理位置 → 自动获取周边 POI
- 精度：引入行业基准数据校验

---

## 4. 模型能力约束

| 能力层级 | 当前状态 | 目标状态 |
|----------|----------|----------|
| 参数提取 | LLM 从自然语言提取 | 标准化输入 + LLM 补全缺失字段 |
| 现实评估 | 规则公式（黑箱算法） | 基于真实订单数据统计 |
| 潜力分析 | 无 | 外部数据 + 行业基准对标 |
| 路径建模 | 固定假设（15%提量等） | 参数化建模 + 概率分布 |
| 报告生成 | LLM 流式生成 | LLM + 数据可视化图表 |

---

## 5. 输出质量约束

1. **数字自洽**：各优化动作的利润增量加总必须等于总利润改善值
2. **事实与建议分离**：报告中必须明确标注哪些是数据事实、哪些是模型建议
3. **假设透明**：所有计算假设必须在报告末尾单独列出
4. **不使用绝对化表述**：禁止"保证""一定"等用语
5. **置信度标注**：潜力分析部分需标注高/中/低置信度
