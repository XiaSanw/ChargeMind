# AI驱动充电桩智能诊断平台 - UI设计文档 v0.1

## 1. 设计目标与整体风格

*   **设计目标**: 为 MVP 演示版本提供一个专业、直观、具有科技感的操作界面，核心在于向受众凸显“AI 语义理解能力”和“数据分析优化价值”，让整个演示过程流畅且具有说服力。
*   **视觉风格**: 建议采用现代企业级 SaaS 风格。推荐使用深色主题（Dark Mode）或清爽的科技蓝白配色。通过大圆角、卡片式阴影和清晰的排版来提升高级感。
*   **布局架构**: 采用经典的**左右分栏布局**，符合人眼“从左至右”的阅读和操作逻辑。左侧负责“输入与过程展示”，右侧负责“结果与报告呈现”。

---

## 2. 核心功能区域划分

界面主要划分为以下四大核心模块：

### 2.1 侧边控制与输入区 (Input & Control Panel)
*   **位置**: 界面左上区域。
*   **元素**:
    *   **非结构化输入框 (Text Edit)**: 宽大、支持多行的文本域，用于用户粘贴或输入场站的自然语言描述。
    *   **快捷填充按钮 (Quick Fill)**: 提供“加载演示案例”功能，点击后一键填入 PRD 中设定的标准话术，极大提高演示效率。
    *   **核心操作按钮 (Primary Action)**: 显眼的“开始智能诊断”按钮。点击后需带有明确的 Loading 动画（如按钮内转圈或进度条），体现系统正在处理。

### 2.2 AI 解析追踪区 (AI Extraction View)
*   **位置**: 界面左下区域（位于输入区下方）。
*   **功能**: 具象化展示“LLM 提取层”的工作成果，让黑盒的 AI 过程变得可见。
*   **展示方式**:
    *   使用清晰的**键值对表格**或**标签卡片**展示提取出的结构化参数（如：场站名称、桩数、电价、利用率等）。
    *   **动效建议**: 数据项可以带有逐个“淡入”或“滑入”的动效，模拟 AI 实时解析提取的过程。

### 2.3 核心数据看板区 (Core Metrics Dashboard)
*   **位置**: 界面右上区域。
*   **功能**: 直击痛点，高亮展示“核心算法层”输出的优化前后对比数据。
*   **视觉元素**:
    *   **关键指标卡片 (KPI Cards)**: 并排展示“当前收益 vs 优化收益”、“当前 IRR vs 优化 IRR”。
    *   **色彩心理学**: 亏损数据使用醒目的红色/橙色，优化后的盈利数据和提升幅度使用绿色，形成强烈的视觉反差。
    *   **可视化图表 (Charts)**: (可选/视进度而定) 引入简单的柱状图对比优化前后的年收益，让数字更直观。

### 2.4 智能诊断报告区 (AI Diagnostic Report Area)
*   **位置**: 界面右下区域（占据最大面积）。
*   **功能**: 呈现“LLM 生成层”输出的最终《降本增效分析报告》。
*   **展示方式**:
    *   **富文本渲染 (Rich Text)**: 报告支持 Markdown 渲染（如各级标题、加粗、列表）。
    *   **结构化呈现**: 清晰划分“现状诊断”、“降本建议”、“增效建议”等模块。
    *   **动效建议**: 采用经典的**“打字机”效果 (Typewriter Effect)** 逐字/逐句输出内容，这是展现大语言模型生成特性的最佳实践。

## 3. 交互流程设计 (User Flow)

1.  **初始空状态 (Idle)**: 界面清爽，输入框留有提示占位符，右侧数据和报告区显示统一的空状态占位图或引导文案。
2.  **数据就绪 (Ready)**: 用户手动输入文字或点击“加载演示案例”，输入框内容被填充。
3.  **处理中状态 (Processing)**:
    *   点击“开始智能诊断”后，按钮禁用。
    *   界面展现全局或局部的分步进度提示：
        *   *步骤 1: 正在呼叫 AI 提取场站参数...*
        *   *步骤 2: 正在运行收益与购电优化算法...*
        *   *步骤 3: AI 正在撰写降本增效诊断报告...*
4.  **结果涌现 (Result Presentation)**:
    *   左下角参数区快速展示提取结果。
    *   右上角数据看板区的数字可加入“数字滚动上升”的动画。
    *   右下角报告区开始“打字机”输出，完成整个演示闭环。

## 4. 针对 PySide6 的技术实现建议

*   **UI 布局框架**: 使用 `QHBoxLayout` 划分左右两大块，再在左右两块中分别使用 `QVBoxLayout` 进行上下堆叠。
*   **样式与美化**: 深度依赖 QSS (Qt Style Sheets) 进行全局样式定义。例如圆角 (`border-radius`)、卡片阴影 (`QGraphicsDropShadowEffect`)、无边框输入框等，彻底摆脱传统 Windows 软件的陈旧感。
*   **防假死机制 (异步多线程)**: 极为关键。必须使用 `QThread` 将模拟的“AI 解析延时”、“算法计算延时”和“打字机输出逻辑”放入后台线程执行。主 UI 线程仅负责接收 `Signal` (信号) 来更新界面组件，确保演示过程中 UI 依然丝滑且响应用户的拖拽、点击等操作。
